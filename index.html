Conceitos de Git e GitHub

Git

Git é um sistema de controle de versão distribuído que permite rastrear mudanças no código fonte durante o desenvolvimento de software. Ele mantém um histórico completo de todas as alterações feitas nos arquivos.

  • Controle de versão local e distribuído
  • Rastreamento de mudanças em arquivos
  • Histórico completo de commits

GitHub

GitHub é uma plataforma de hospedagem de código que utiliza Git para controle de versão. Oferece recursos colaborativos como pull requests, issues e wikis para facilitar o trabalho em equipe.

  • Hospedagem de repositórios Git
  • Ferramentas de colaboração
  • Interface web intuitiva

Comandos Básicos do Git

git init

Inicializa um repositório Git

git add .

Adiciona arquivos ao staging

git commit -m "mensagem"

Cria um commit com mensagem

git push

Envia commits para o repositório remoto

git pull

Baixa e mescla mudanças do remoto

git status

Mostra o status dos arquivos

Fluxo de Git/GitHub

Fluxo de Trabalho Típico

Entenda o processo completo desde a criação até a publicação do código

1

Clone ou Fork

Clone o repositório ou faça um fork para sua conta

git clone <url-do-repositorio>
2

Criar Branch

Crie uma nova branch para sua feature ou correção

git checkout -b nome-da-branch
3

Fazer Alterações

Edite os arquivos necessários e teste suas mudanças

4

Commit

Adicione e faça commit das suas alterações

git add . git commit -m "Descrição clara da mudança"
5

Push

Envie sua branch para o GitHub

git push origin nome-da-branch
6

Pull Request

Crie um Pull Request no GitHub para revisar e mesclar suas mudanças

Boas Práticas

  • Use mensagens de commit descritivas
  • Faça commits pequenos e frequentes
  • Sempre teste antes de fazer push
  • Use branches para features
  • Mantenha o histórico limpo
  • Documente mudanças importantes

Como Resolver Conflitos

O que são Conflitos?

Conflitos ocorrem quando o Git não consegue automaticamente mesclar mudanças de diferentes branches ou commits. Isso acontece quando as mesmas linhas de código foram modificadas de formas diferentes.

Situações Comuns:
  • • Duas pessoas editaram a mesma linha
  • • Um arquivo foi deletado em uma branch e modificado em outra
  • • Mudanças conflitantes em merge ou rebase

Passo a Passo para Resolver

1

Identificar o Conflito

O Git mostrará uma mensagem de erro indicando arquivos em conflito

git status
2

Abrir o Arquivo

Abra o arquivo conflitante em seu editor. Você verá marcadores como:

<<<<<<< HEAD
Seu código atual
=======
Código da outra branch
>>>>>>> nome-da-branch
3

Resolver Manualmente

Edite o arquivo removendo os marcadores e mantendo o código correto

4

Marcar como Resolvido

Adicione o arquivo resolvido ao staging

git add nome-do-arquivo
5

Finalizar

Complete o merge ou continue o rebase

git commit
